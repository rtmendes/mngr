# Issue 1: _is_transient_ssh_error docstring missing ChannelException

## What and where

The `_is_transient_ssh_error` function at `libs/mng/imbue/mng/hosts/host.py` line 102 has a docstring that lists the exception types it matches but does not mention `ChannelException`.

## Why it is a problem

This branch added dedicated `ChannelException` handling in `_get_file_with_transient_retry` and `_put_file_with_transient_retry` that differs from the general `SSHException` handler (ChannelException does NOT disconnect, while SSHException does). The `_is_transient_ssh_error` function covers ChannelException implicitly since it's a subclass of SSHException, but the docstring should mention this explicitly since it's now a significant case with specialized handling.

## Planned fix

Add `ChannelException` to the docstring's "Matches:" list, noting that it's a subclass of SSHException.

## Risks

None -- this is a documentation-only change.
