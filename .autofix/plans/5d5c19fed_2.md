# Issue 2: Missing assertion on created_branch_name in test

## What and where
In `libs/mng/imbue/mng/api/test_create.py`, the `test_worktree_with_existing_branch` test
(starting at line 403) checks that the worktree was created on the correct branch, but does
not assert that `created_branch_name` is `None` on the agent.

## Why it is a problem
When an existing branch is checked out (rather than a new branch being created),
`created_branch_name` must be `None`. This is because `mng destroy --remove-created-branch`
uses `created_branch_name` to determine whether to delete the branch. If it were incorrectly
set to the existing branch name, destroying the agent would delete a branch the user created
independently of mng. Without an assertion on this property, a regression could go undetected.

## Planned fix
Add an assertion in the test that verifies `agent.get_created_branch_name() is None` after
creating the worktree with an existing branch.

## Risks
Minimal -- adding an assertion to an existing test.
