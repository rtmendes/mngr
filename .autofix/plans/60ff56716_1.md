# Issue 1: Missing test for stale symlink replacement

## What the issue is and where

File: `libs/mng/imbue/mng/hosts/host_test.py`, after line 2442 (end of work_dir_extra_paths tests)

The `_apply_work_dir_extra_paths` method's batched shell script handles stale symlink replacement
(where a symlink exists at the target but points to the wrong source), but there is no unit test
covering this scenario. This behavior was introduced in commit 304b7b2ac and preserved in the
batched rewrite, but a regression would go undetected.

## Why it is a problem

The stale symlink replacement is handled by the `readlink -f` comparison in the `elif` branch of
the batched shell script. This is a non-trivial piece of shell logic that could easily break
without test coverage. The existing idempotent test only covers the case where the symlink points
to the CORRECT target.

## Planned fix

Add a new test `test_apply_work_dir_extra_paths_share_same_host_replaces_stale_symlink` that:
1. Creates source `.venv` directory
2. Creates a symlink at `work_dir/.venv` pointing to a DIFFERENT location (stale symlink)
3. Calls `_apply_work_dir_extra_paths` with SHARE mode
4. Asserts the symlink now points to the correct source

Use the existing `source_and_work_dirs` fixture and `local_host` fixture.

## Risks and edge cases

- The test should work on both macOS and Linux (readlink -f behavior is consistent for non-broken symlinks on both).
- The test relies on the shell's readlink -f working correctly for local execution via pyinfra.
