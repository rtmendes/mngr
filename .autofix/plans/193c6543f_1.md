# Issue 1: Uncommitted changes to error message handling

## What the issue is and where

There are unstaged changes in `libs/mngr/imbue/mngr/hosts/host.py` (lines 1618-1627) and `libs/mngr/imbue/mngr/api/test_create.py` (line 502) that improve the "already checked out" error message handling but have not been committed.

## Why it is a problem

The changes are coherent improvements: the error message now uses git's own stderr message (which includes the actual branch name and the worktree path) instead of constructing a message that only names the branch. The test is updated to match the new message format. Leaving these uncommitted means the branch is in an incomplete state.

## Planned fix

Commit the unstaged changes to both files as a single commit with a clear message explaining the improvement.

## Risks or edge cases

The `git_msg` extraction (`stderr.strip().split("\n")[-1].removeprefix("fatal: ").strip()`) could pick up an unexpected line if git's stderr format changes, but this is a reasonable heuristic and the fallback MngrError would still fire for truly unexpected errors.
